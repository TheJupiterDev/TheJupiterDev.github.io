Why I Build Tools No One Asked For

May 6, 2025

No one asked me to build mDirt.

No one requested a Minecraft datapack generator powered by PySide6, with features I personally find useful and probably no one else even knows about.

And that’s exactly why I built it.

🛠 The Joy of Scratching an Itch

Some tools are born out of user demand. Others are born because something in your brain says, “Wait — wouldn’t it be cool if…”

I love those projects. The ones that solve a problem only I seem to have. The ones that don’t need approval, or market validation, or even a single user besides me.

There’s no roadmap. No meetings. Just curiosity and a keyboard.

🌱 Building Just to Learn

Sometimes the project doesn’t go anywhere. Sometimes it *shouldn’t* go anywhere.

I once built a command-line tool that converted markdown notes into a fictional-world database. I never used it beyond week one. But it taught me regex, file I/O, and how to clean up spaghetti code with modular thinking.

Another time, I created a pixel-art grid generator in Python. For no reason. Just vibes. And I learned more about canvas rendering in two days than I had in months.

The point wasn’t the product. It was the process.

💡 When Usefulness Sneaks Up on You

The funny part is: sometimes these solo tools actually end up helping people.

mDirt started as a personal challenge. Now it's a thing I update, polish, and occasionally share — because it turned out *someone else* had the same itch.

I've had friends use stuff I built in a weekend, things I almost didn't finish, and tell me it saved them time. That’s always wild. Because it proves that making something for *yourself* first isn’t selfish — it’s sustainable.

🚫 Why I Don’t Wait for Permission

If I waited for someone to tell me what to build, I probably wouldn’t build anything at all.

I make stuff no one asked for because that’s where my best ideas come from. That’s where I experiment, play, and occasionally surprise myself.

When you stop chasing an audience and start chasing curiosity, you build with less fear and more fun.

☕ What I’ll Keep Making

Tools for fictional timelines? Sure.

Apps that automate things only I care about? Absolutely.

Weird hybrids of writing, coding, and worldbuilding logic puzzles? Sign me up.

Because even if no one asks for it — even if no one else ever uses it — I’ll still make it.

And honestly, that’s the point.

— Joel
